## 개요

기기 목록 및 실시간 상태 조회 시 정렬 옵션을 추가하고, 특정 기준에 따른 정렬 로직을 구현하였습니다. 또한 예약 생성 서비스의 코드 스타일을 수정하여 가독성을 개선하였습니다.

## 본문

### 작업 이유
사용자가 기기 현황을 파악하기 쉽도록 층별, 종류별로 정렬된 데이터를 제공할 필요가 있었습니다. 또한 기존 코드의 가독성을 높이기 위해 일부 서비스 로직의 스타일 수정을 진행하였습니다.

### 구현 방식
- 관리자용 기기 목록 조회 및 일반 사용자용 전체 기기 현황 조회 API에 정렬 여부를 결정하는 `sorted` 파라미터를 추가하였습니다.
- QueryDSL 및 Spring Data JPA의 `Sort` 기능을 활용하여 층(오름차순) → 기기 종류(세탁기 우선) → 위치(왼쪽 우선) → 번호(오름차순) 순서로 정렬되도록 구현하였습니다.
- `CreateReservationServiceImpl`에서 인증 정보로부터 사용자 ID를 가져오는 코드의 줄바꿈을 수정하였습니다.

### 현재 상태
기기 관련 API 호출 시 기본적으로 정렬된 데이터를 반환하며, 필요한 경우 `sorted=false`를 통해 정렬되지 않은 데이터를 요청할 수 있습니다. 예약 서비스의 코드가 팀 컨벤션에 맞춰 정돈되었습니다.
